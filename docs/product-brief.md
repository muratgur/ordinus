# Ordinus Product Brief

## Executive Summary

Ordinus is a desktop workspace for coordinating AI agents that help with software work. It gives users a single place to define agents, assign work, follow progress, review outputs, and keep a useful history of what happened.

The product is for people who want more than a one-off chat with an AI assistant. Ordinus helps turn AI agents into a small working team: each agent can have a role, a responsibility, and a place in the workflow.

## Product Vision

Ordinus should feel like a calm command center for AI-assisted work. A user should be able to open the app, choose what they want done, select or create the right agents, and understand the state of the work without digging through scattered terminals, notes, or chat histories.

The experience should be practical, clear, and trustworthy. Users should always know:

- Which agents exist.
- What each agent is responsible for.
- What work is planned, running, blocked, or finished.
- What outputs were produced.
- What needs their attention.

## Target Users

Ordinus is built for developers, technical founders, product builders, and advanced users who use AI tools as part of their daily work.

These users want to:

- Organize AI assistance around real work, not isolated prompts.
- Split larger goals into smaller tasks.
- Use specialized agents for planning, implementation, review, testing, or research.
- Preserve context and outputs across a project.
- Keep control over decisions and approvals.

## Core Product Principles

1. Work should be visible  
   Users should be able to see what is happening, what happened before, and what remains to be done.

2. Agents should have clear roles  
   Ordinus should make it easy to create specialized agents with understandable responsibilities.

3. Coordination matters  
   The product should help agents work together instead of simply running separate conversations.

4. The user stays in control  
   Agents can suggest, draft, plan, and execute, but the user should always be able to inspect, pause, redirect, or approve important work.

5. Simple before powerful  
   The early product should focus on a small set of reliable workflows before adding broad integrations or complex automation.

## Initial Product Focus

The first useful version should help users:

- Create and manage agents.
- Start conversations with individual agents.
- Create work items from a goal or request.
- Assign work to agents.
- Track work status and progress.
- Review agent outputs and summaries.
- Keep a persistent project history.

Advanced collaboration, external integrations, and marketplace-like features can come later.

## What Ordinus Is Not

Ordinus is not just another chat interface. It is also not meant to hide agent activity behind a black box. The product should make AI work more organized, observable, and reusable.

It should avoid becoming overly complicated too early. The value is not in having many buttons or integrations; the value is in helping users reliably coordinate AI agents around meaningful work.

## Success Definition

Ordinus is on track when a user can open the app, create a few useful agents, give them a real project goal, follow the resulting work, review the outputs, and return later with the project context still clear.
